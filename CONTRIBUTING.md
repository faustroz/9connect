# Contributing

Thanks for helping improve this project.

## Workflow

1. Open an issue for bugs or feature ideas.
2. Fork the repository.
3. Create a small focused branch.
4. Run the documented setup and checks.
5. Open a pull request with a clear summary and screenshots/logs when useful.

## Pull Request Checklist

- Keep changes focused.
- Avoid committing secrets, generated files, or local account data.
- Update documentation when behavior changes.
- Explain testing done.
